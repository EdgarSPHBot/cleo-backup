## Bug Caught By Hannah (Apr 1 evening)

- **Detection:** Python div balance check (`open count - close count = 1`)
- **Fix:** Added the missing closer before the legacy div block
- **Lesson:** When using `edit` to replace HTML chunks with structural changes, **always run a div balance check** as a final step. JS syntax check is not sufficient — HTML layout damage manifests at runtime only.

## Recommended balance check command
```python
python3 -c "
text = open('FILE').read()
opens = text.count('<div')
closes = text.count('</div>')
print(f'<div: {opens}, </div>: {closes}, diff: {opens-closes}')
"
```

## Brave Search Fix (May 2 early UTC)
Web search was randomly failing because the gateway was load-balancing across MiniMax/Gemini/Brave plugins, all defaulting on. MiniMax and Gemini had no keys configured.

**Resolution:**
- `openclaw config set tools.web.search.provider brave`
- `openclaw config set plugins.entries.minimax.enabled false`
- `openclaw config set plugins.entries.google.enabled false`
- Brave key wired via `plugins.entries.brave.config.webSearch.apiKey` (ref file `/home2/cleo/brave_search.txt`) — `BRAVE_API_KEY` env var also exists
- **Hard restart required** via `systemctl --user restart openclaw-gateway.service` — SIGUSR1 hot-reload doesn't reload plugin enablement

**Brave free tier limits:** 1 req/sec, 2000/month. Run web_search calls sequentially, not in parallel, to avoid 429s.

## Site Down → Resurrect Lesson (May 2 ~01:00 UTC)
After `systemctl --user restart openclaw-gateway.service`, the cadence prototype site at port 8765 went down. Root cause: PM2 daemon shares the user systemd session and got restarted too, losing its in-memory process list.

**Fix:** `pm2 resurrect` (reads from `~/.pm2/dump.pm2` which had been saved earlier).

**Lesson:** Whenever I hard-restart the gateway via systemd, **always run `pm2 status` afterwards**. If cadence is missing from the list, run `pm2 resurrect`. The dump file is the single source of truth for PM2's saved process list.

To prevent this entirely, would need either (a) PM2 startup script that David has to sudo enable, or (b) move cadence under systemd directly with its own service unit. Neither in scope tonight.
